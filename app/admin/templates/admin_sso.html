{{ icon('lock', size=22) }} SSO 單一登入（OIDC / SAML）

SSO 是附加登入方式，與本機 / LDAP / AD 並存 —— 啟用後本機 jtdt-admin 仍可登入當作 break-glass。首次以 SSO 登入會自動建立帳號（預設「一般使用者」角色）；IdP 群組會同步成本站群組，可在「權限矩陣」對應到角色。

OIDC

欄位Microsoft 365 / Entra IDGoogle WorkspaceKeycloak
Issuer https://login.microsoftonline.com/<tenant-id>/v2.0 https://accounts.google.com https://<keycloak>/realms/<realm>
Client ID / Secret 在 IdP 端建立應用程式（Entra：應用程式註冊；Google：OAuth 用戶端 ID；Keycloak：confidential client）後取得
Scopesopenid email profile（三家相同）
使用者名稱 claim preferred_username email（Google 無 preferred_username） preferred_username
Email / 顯示名稱 claimemail / name（三家相同）
群組 claim groups（需在 token 設定加 groups claim） —（Google id_token 不含群組，留空，角色用權限矩陣手動指派） groups（需加 Group Membership mapper）

⚠ Entra 群組預設送的是群組 GUID 不是名稱；「管理員群組」與權限矩陣請填該 GUID，或在 Entra 設定送 group name。
⚠ Google 沒有標準群組 claim；要群組對應需另接 Cloud Identity，否則留空、用權限矩陣手動給角色。

SAML

欄位Entra ID（企業應用程式）Keycloak
IdP EntityID https://sts.windows.net/<tenant-id>/ https://<keycloak>/realms/<realm>
IdP SSO URL https://login.microsoftonline.com/<tenant-id>/saml2 https://<keycloak>/realms/<realm>/protocol/saml
簽章憑證從 IdP 的 Federation Metadata / SAML descriptor 取 X.509（貼憑證內容，不含 BEGIN/END 行）
群組屬性 http://schemas.microsoft.com/ws/2008/06/identity/claims/groups groups（需加 Group list mapper）

SAML 也是 Entra 送群組 GUID；Google Workspace SAML 不送群組屬性。把本站 SP metadata（<base>/auth/saml/metadata）餵給 IdP 最省事。
